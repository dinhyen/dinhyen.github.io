How to grant permissions to the Application Pool Identity for IIS 7.5

Up to IIS 7 (Windows Vista and Windows Server 2008), the default app pool identity is NetworkService 7. In IIS 7.5 (Windows 7 and Windows Server 2008 R2) the app pool identity is changed to something called ApplicationPoolIdentity. Let’s say you set up a virtual directory or application that uses this app pool. You’ll need to grant access permission on the physical path to ApplicationPoolIdentity. However, you’d find that there is no such user account.
