Using Angular root scope for communication between controllers

Angular makes it simple to create standard index and details views. Using the ubiquitous task list as an example, I can set up routes using the $route service as follows: $routeProvider .when('/tasks', { templateUrl: '_tasks.html', controller: 'TasksIndexCtrl' }) .when('/task/:id', { templateUrl: '_task.html', controller: 'TaskDetailsCtrl' }) .otherwise({ redirectTo: '/tasks' }); } The routes definitions specify the Angular controller and HTML template to use when the user navigates to a particular path. Here we have 2 controllers, one for the index page and one for the details page.
